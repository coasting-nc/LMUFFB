# MoTeC i2 Pro File Format Specification

This specification is derived from reverse-engineered community efforts (ldparser, Aim_2_MoTeC) and documented research.

## File Types

- **.ld**: Proprietary binary file containing the main header, channel metadata, and raw telemetry data.
- **.ldx**: XML sidecar file containing session metadata, lap markers (beacons), and local mathematical channel definitions.
- **.idx**: Binary index file generated by MoTeC i2 Pro for faster loading. This should NOT be generated by third-party tools.

## .ld Binary Structure (Little-Endian)

### 1. Main Header (0x664 bytes)

| Offset | Size | Type | Description |
| :--- | :--- | :--- | :--- |
| 0x00 | 4 | u32 | **File Marker**: 0x40 (64) |
| 0x08 | 4 | u32 | **Channel Metadata Pointer**: Absolute offset to first channel header. |
| 0x0C | 4 | u32 | **Channel Data Pointer**: Absolute offset to telemetry data block. |
| 0x46 | 4 | u32 | **Device Serial**: Unique identifier (e.g., 12345). |
| 0x4A | 8 | str | **Device Type**: e.g., "ADL" (null-padded). |
| 0x52 | 2 | u16 | **Device Version**: e.g., 420. |
| 0x56 | 4 | u32 | **Number of Channels**: Total count. |
| 0x5E | 16 | str | **Session Date**: "DD/MM/YYYY" (null-padded). |
| 0x7E | 16 | str | **Session Time**: "HH:MM:SS" (null-padded). |
| 0x9E | 64 | str | **Driver Name** (null-padded). |
| 0xDE | 64 | str | **Vehicle ID** (null-padded). |
| 0x15E | 64 | str | **Venue Name** (null-padded). |
| 0x5DE | 4 | u32 | **Pro Logging Flag**: **0xc81a4** (unlocks Pro features). |
| 0x624 | 64 | str | **Short Comment** (null-padded). |

### 2. Channel Metadata Header (Linked List)

Each channel is defined by a 0x70 byte block.

| Offset | Size | Type | Description |
| :--- | :--- | :--- | :--- |
| 0x00 | 4 | u32 | **Next Header Pointer**: Absolute offset to next channel (0 if last). |
| 0x04 | 4 | u32 | **Data Pointer**: Absolute offset to this channel's data array. |
| 0x08 | 4 | u32 | **Number of Samples**: Total samples in this channel. |
| 0x0C | 2 | u16 | **Unknown**: Usually 0x0001. |
| 0x0E | 2 | u16 | **Data Type**: 0x0003 (i16), 0x0005 (i32), 0x0007 (f32). |
| 0x10 | 2 | u16 | **Sample Size**: 2 (i16), 4 (i32/f32). |
| 0x12 | 2 | u16 | **Sample Rate**: Hz. |
| 0x14 | 2 | i16 | **Scale Multiplier**: Applied to raw value. |
| 0x16 | 2 | i16 | **Scale Divisor**: Applied to raw value. |
| 0x18 | 2 | i16 | **Scale Offset**: Applied to raw value. |
| 0x1A | 2 | i16 | **Decimal Places**: Hint for UI display. |
| 0x1C | 32 | str | **Channel Name** (null-padded). |
| 0x3C | 8 | str | **Short Name** (null-padded). |
| 0x44 | 12 | str | **Unit Name** (null-padded). |

**Scaling Formula (Integer to Float):**
`displayed_value = (raw_value * multiplier / divisor / (10^decimal_places)) + offset`

### 3. Telemetry Data Block

The data for each channel is stored contiguously. While the MoTeC format supports interleaved data, the pointer-based architecture also allows for channel-by-channel blocks.

## .ldx XML Structure

The XML file provides high-level metadata and lap markers.

```xml
<?xml version="1.0" encoding="utf-8"?>
<LDXFile ValidationCode="0" Version="1.0">
  <Layers>
    <Layer>
      <MarkerGroup Name="Beacons" Index="0">
        <Marker Version="100" EntityID="1" Time="0.000000"/>
        <Marker Version="100" EntityID="1" Time="125.432000"/>
      </MarkerGroup>
    </Layer>
  </Layers>
  <Details>
    <String Id="Driver" Value="John Doe"/>
    <String Id="Vehicle" Value="LMP2"/>
    <String Id="Venue" Value="Le Mans"/>
  </Details>
</LDXFile>
```

- **Marker**: `Time` is in seconds from the start of the log.
- **Details**: String IDs match those in MoTeC's internal database (Driver, Vehicle, Venue, Event, etc.).
