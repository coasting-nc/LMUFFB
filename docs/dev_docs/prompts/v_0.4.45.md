**Context:**
The user wants two usability improvements:
1.  An option to keep the LMUFFB window visible over the game ("Always on Top").
2.  The ability to fine-tune slider values using the keyboard arrow keys while hovering over them, without needing to click/drag.

**Implementation Requirements:**

1.  **Configuration (`src\Config.h`, `src\Config.cpp`)**:
    *   Add `static bool m_always_on_top` to the `Config` class.
    *   Initialize it to `false`.
    *   Update `Config::Save` and `Config::Load` to persist this setting to `config.ini` using the key `always_on_top`.

2.  **GUI Layer (`src\GuiLayer.cpp`)**:
    *   **Helper Function**: Implement `void SetWindowAlwaysOnTop(HWND hwnd, bool enabled)` using `SetWindowPos` and `HWND_TOPMOST` / `HWND_NOTOPMOST`.
    *   **Initialization**: In `GuiLayer::Init`, apply the saved `m_always_on_top` setting immediately after window creation.
    *   **Tuning Window**:
        *   Add a checkbox "Always on Top" near the top of `DrawTuningWindow`.
        *   When toggled, call `SetWindowAlwaysOnTop` immediately.
    *   **Keyboard Logic**:
        *   Modify the `FloatSetting` lambda in `DrawTuningWindow`.
        *   Add logic: `if (ImGui::IsItemHovered())`.
        *   Inside that block, check `ImGui::IsKeyPressed(ImGuiKey_LeftArrow)` and `ImGui::IsKeyPressed(ImGuiKey_RightArrow)`.
        *   Adjust the value by a small step (`0.01` for small ranges, `0.5` for ranges > 50.0).
        *   Ensure the value is clamped to `min`/`max` and `selected_preset` is set to `-1` (Custom) if changed.
        *   Add a tooltip explaining the feature: "Fine Tune: Arrow Keys".
